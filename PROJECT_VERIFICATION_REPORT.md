### ✅ 8. Retrieve Users by Province (4 Marks)

**Status**: PASSED ✅

**Implementation**: Two methods - by code AND by name

**File**: `UserRepository.java`
```java
@Query("SELECT u FROM User u WHERE u.village.cell.sector.district.province.code = :provinceCode")
Page<User> findUsersByProvinceCode(@Param("provinceCode") String provinceCode, Pageable pageable);

@Query("SELECT u FROM User u WHERE u.village.cell.sector.district.province.name = :provinceName")
Page<User> findUsersByProvinceName(@Param("provinceName") String provinceName, Pageable pageable);
```

**Endpoints**:
- `GET /api/users/province/code/{code}` - Search by province code
- `GET /api/users/province/name/{name}` - Search by province name

**Query Logic**:
1. Starts with User entity (u)
2. Navigates through relationships: u.village → cell → sector → district → province
3. Filters by province code or name
4. Hibernate generates SQL with JOINs
5. Returns paginated results

**Generated SQL** (approximate):
```sql
SELECT u.* FROM user_table u
INNER JOIN village v ON u.village_id = v.id
INNER JOIN cell c ON v.cell_id = c.id
INNER JOIN sector s ON c.sector_id = s.id
INNER JOIN district d ON s.district_id = d.id
INNER JOIN province p ON d.province_id = p.id
WHERE p.code = ? -- or p.name = ?
LIMIT 10 OFFSET 0;
```

**Explanation for Viva**:
"We use JPQL (Java Persistence Query Language) to traverse the relationship chain from User to Province. The query 'u.village.cell.sector.district.province.code' tells Hibernate to follow the foreign key relationships. Hibernate automatically generates the necessary JOIN statements. This allows us to retrieve all users from a specific province without storing redundant province data in the user table."
